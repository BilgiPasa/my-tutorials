Linux kullanıyorsan şunları yap
{
    Başlamadan önce, kendi distrona göre olan yapılacaklar listeni ve Linux'ta kurulacak uygulamalar listeni
    takip et. Oradaki işlerin bitince buradan devam et.

    .NET'i, şu sitedeki yönergelere uyarak sistemine kur:
        https://dotnet.microsoft.com/en-us/download

    Ardından terminale şu komutları gir
    {
        Sistemin Debian tabanlı ise
        {
            sudo apt update
            sudo apt-get install python3-tk python3-pip
            sudo apt install default-jre default-jdk texlive-full build-essential python3-dev libcairo2-dev libpango1.0-dev ffmpeg
        }

        Sistemin Fedora tabanlı ise
        {
            sudo dnf update
            sudo dnf swap ffmpeg-free ffmpeg --allowerasing
            sudo dnf install java-latest-openjdk.x86_64 java-latest-openjdk-devel.x86_64 python3-tkinter texlive-scheme-full python3-devel pkg-config cairo-devel pango-devel python3-pip
        }
    }

    (Not: Bilgisayarında birden çok Java versiyonu yüklüyse ve bunları yönetmek istiyorsan
    "sudo alternatives --config java" komutunu gir. Buradaki versiyonlardan ana olarak kullanmak istediğini
    seç. Eğer ki eksi bir Java versiyonunu silmek istiyorsan silmemeni öneririm çünkü o Java versiyonuna bağlı
    olan kullandığın uygulama veya paket olabilir.)

    Debian'da Java kurulum komutunun kaynağı: https://wiki.debian.org/Java
    Debian'da FFmpeg kurulum komutunun kaynağı: https://www.geeksforgeeks.org/linux-unix/how-to-install-ffmpeg-in-linux/
    Fedora'da Java kurulum komutunun kaynağı: https://docs.fedoraproject.org/en-US/quick-docs/installing-java/
    Fedora için FFmpeg komutunun kaynağı (Fedora 37 başlıklı kısma bak): https://rpmfusion.org/CommonBugs
    Tkinter (tk) kurulum komutlarının kaynağı: https://stackoverflow.com/questions/76105218/why-does-tkinter-or-turtle-seem-to-be-missing-or-broken-shouldnt-it-be-part
    PIP kurulum komutlarının kaynağı: https://www.geeksforgeeks.org/python/how-to-install-pip-in-linux/
    Diğer komutları ise Manim Community'nin sayfasından baktım da yazdım.

    Texlive'ı, Manim'de LaTeX'i kullanabilmek için kuruyorum.

    Python dosyalarını kullanacaksan .venv ile kullan. .venv kurmak için internete bağlan, F1'e bas ve arama
    yerine "Python: Ortam Oluşturun" veya "Python: Create Environment" yaz ve o seçeneğe tıkla. Çıkan menüde
    Venv'yi seç ve yorumlayıcı olarak sağında "Genel" yazanı seç. Ardından VS Code'u KAPAT VE AÇ. Sonrasında
    bir Python dosyasını çalıştır. Sağ altta yorumlayıcı olarak .venv kullanıldığını görüyorsan .venv çalışıyor
    demektir. (Nolur nolmaz Python ile ilgili bilgisayarın kendi terminalinde bir şey kurarsan veya silersen,
    .venv sanal ortamını sil ve yeniden oluştur.) (Ve eğer ki kuracağın şeyi .venv'ye değil de bilgisayara
    kurarsan o kurduklarını "pip3 uninstall paket_adı" yazarak sil ve o şeyleri .venv'de kur. Bir problem
    çıktığı için değil, nolur nolmaz bir problem çıkmaması için bunu yapmanı öneriyorum.)

    Yukarıdaki yönergeleri takip ettikten sonra, VS Code'daki sanal alan içindeki terminalde şu komutları gir:
        pip3 install manim speedtest-cli pygame-ce ursina
}

Windows kullanıyorsan şunları yap
{
    Başlamadan önce, Windows kurduktan sonra yapılacaklar listeni ve Windows'ta kurulacak uygulamalar listeni
    takip et. Oradaki işin bitince buradan devam et.

    .NET SDK'in kurulum dosyalarını şu siteden indir ve kur:
        https://dotnet.microsoft.com/en-us/download

    Eğer ki; C# veya Unity, .NET Framework'ü kurmanı istiyorsa kurulum dosyalarını şu siteden indir ve kur:
        https://dotnet.microsoft.com/en-us/download/dotnet-framework

    Python'un sitesine girip Windows için en yukarıdaki stabil versiyonunun kurulum dosyasını indir ve kur.
    Kurarken Python'un kurulum ekranında çıkan "Python'u PATH'e ekle" ve "Yönetici olarak çalıştır" gibi şeyler
    yazan o iki seçeneği işaretle ve öyle sistemine kur. Ardından terminale şu komutu gir:
        winget install ffmpeg

    Java'yı kurmak için şu https://www.oracle.com/java/technologies/downloads/ siteye gir ve Windows MSI
    Installer ile Java'yı sistemine kur. (exe'yi de kullanabilirsin ama msi daha güvenli diye biliyorum.)

    Java'nın eski sürümlerini silmek için Windows arama kısmına "Program ekle veya kaldır" yaz ve çıkan
    pencereden Java'nın eski sürümlerini sil. Windows'ta sadece JDK kurman yeterli bu arada. Yani Windows'ta
    ayrıca JRE kurmana gerek yok.

    MiKTeX'i şu siteden https://miktex.org/download normal bir şekilde sistemine kur.
    (MiKTeX'i, Manim'de LaTeX'i kullanabilmek için kuruyorum.)

    Python dosyalarını kullanacaksan .venv ile kullan. .venv kurmak için internete bağlan, F1'e bas ve arama
    yerine "Python: Ortam Oluşturun" veya "Python: Create Environment" yaz ve o seçeneğe tıkla. Çıkan menüde
    Venv'yi seç ve yorumlayıcı olarak sağında "Genel" yazanı seç. Ardından VS Code'u KAPAT VE AÇ. Sonrasında
    bir Python dosyasını çalıştır. Sağ altta yorumlayıcı olarak .venv kullanıldığını görüyorsan .venv çalışıyor
    demektir. (Nolur nolmaz Python ile ilgili bilgisayarın kendi terminalinde bir şey kurarsan veya silersen,
    .venv sanal ortamını sil ve yeniden oluştur.) (Ve eğer ki kuracağın şeyi .venv'ye değil de bilgisayara
    kurarsan o kurduklarını "pip3 uninstall paket_adı" yazarak sil ve o şeyleri .venv'de kur. Bir problem
    çıktığı için değil, nolur nolmaz bir problem çıkmaması için bunu yapmanı öneriyorum.)

    Yukarıdaki yönergeleri takip ettikten sonra, VS Code'daki sanal alan içindeki terminalde şu komutları gir:
        python -m pip install manim
        pip3 install ursina speedtest-cli pygame-ce
}

C# veya Java ile ilgili şeyler yapacaksan proje oluşturarak yap.

C# projesi oluşturacaksan yapılacaklar
{
    İlk önce kendi oluşturacağın projenin adında boş bir klasör oluştur.

    Ardından o oluşturduğun boş klasörü VS Code ile aç. Sonra F1'e tıkla ve ".NET: Yeni Proje"
    (yani ".NET: New Project") olan seçeneği seç.

    Proje "Konsol Uygulaması" olsun (yani "Console App").

    Proje adını ver ve proje konumu varsayılan ne ise o olsun.

    Çözüm dosyası biçimi seçeneği çıkarsa .slnx'i seç çünkü .sln seçersen sıkıntı çıkabilir.

    Ardından projeyi oluşturmadan önce "Tüm şablon seçeneklerini göster" kısmını seç.

    Oradan "Üst düzey deyimler kullanmayın" seçeneğini aç ve "True" yap.

    Ardından projeyi oluştur.
}

Java projesi oluşturacaksan yapılacaklar
{
    Projeyi oluşturmak istediğin yerde VS Code'u aç ve F1'e tıkla.

    Çıkan menüde "Java: Create Java Project" seçeneğini seç.

    Proje hali olarak "No Build Tools" seçeneğini seç.

    Ardından ekrana çıkan proje kısmı seçme menüsünden (eğer ki o menüde çıkan dizin senin VS Code'da açmış
    olduğun dizin ise) bir şeyi seçmeden direk "Select the project location"a tıkla. (Öyle değilse VS Code'da
    açmış olduğun dizine gel.)

    Ardından Java projene bir ad ver.

    Yeni bir pencere açılarak senin Java projeni göstermesi lazım.

    Oluşturduğun Java projende .java dosyaların (evet, aynı projeye birkaç tane .java dosyası koyabilirsin)
    "src" isimli klasörün içinde olması lazım. .java dosyalarının adı ne istersen olabilir. Yani ismi "App"
    olmak zorunda değil ama Class adın ile dosya adının uyuştuğuna emin ol.

    Bu arada Java projesinin içineki "src" klasörünün içine başka bir yerden aldığın .java dosyasını koyacaksan
    (genellikle ilk satırda olan) "package" satırını sil. Eğer ki VS Code projen özel bir package kullanıyor
    ise o package satırındaki package adını VS Code projendeki package adı ne ise o yap.
}

Not1:
    VS Code'da run and debug butonunu gizleyip geri açamıyorsan sağ üstte üç nokta kısmına sağ tık yap ve "run
    and debug" seçeneğine tıkla.

Not2:
    Aynı python sanal ortamını hem Linux'ta hem de Windows'ta kullanamazsın. O yüzden hangi işletim sisteminde
    python ile uğraşacaksan ona göre kur.

Not3:
    Eğer ki Python'da yapmış olduğun bir projenin dosyalarını bir yere aktaracaksan ve o projende .venv
    klasörü varsa, .venv klasörünü aktarma. Diğer dosyaları aktar .venv'yi aktarma. .venv klasörünü kurmak
    için yukarıda anlattığım yönergeleri takip et.

Not4:
    Eğer ki Python'da yapmış olduğun ve GitHub'da yönettiğin bir projen varsa ve o projende .venv klasörü
    varsa, o projenin root klasöründe bir ".gitignore" dosyası oluştur. .gitignore dosyasının içine ise
    .venv klasörün nerede ise konumunu yaz ve yanına bir eğik çizgi (/) koy. Örneğin .venv klasörün o
    projenin root klasöründe ise .gitignore dosyasının içine şunu yaz: .venv/

    .gitignore dosyası ile .venv klasörünü ihmal ettiğimiz için o projeyi clone'ladığında .venv klasörü gelmez.
    .venv klasörünü kurmak için yukarıda anlattığım yönergeleri takip et.
